# Quiver

Advanced mathematics for cybersecurity, presented as validated Haskell and Python lessons published to a Quarto wiki on GitHub Pages.

---

## Prerequisites

| Tool | Install |
|------|---------|
| GHC (`runghc`) | `curl --proto '=https' --tlsv1.2 -sSf https://get-ghcup.haskell.org \| sh` |
| uv | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| just | `brew install just` |
| Quarto | https://quarto.org/docs/get-started/ |
| gh CLI | `brew install gh` then `gh auth login` |

After installing GHC, activate the environment or open a new shell so `runghc` is on your PATH.

---

## Quick Start: Creating a Lesson

```bash
# 1. Scaffold
just new-lesson foundations modular-arithmetic

# 2. Implement (edit the two source files)
#    lessons/foundations/modular-arithmetic/lesson.hs
#    lessons/foundations/modular-arithmetic/lesson.py

# 3. Validate
just validate lessons/foundations/modular-arithmetic/lesson.hs
just validate lessons/foundations/modular-arithmetic/lesson.py

# 4. Export to LaTeX
just export-latex lessons/foundations/modular-arithmetic

# 5. Author the Quarto page
#    lessons/foundations/modular-arithmetic/lesson.qmd

# 6. Preview locally
just preview

# 7. Publish
just publish-gist lessons/foundations/modular-arithmetic/lesson.hs
just publish-gist lessons/foundations/modular-arithmetic/lesson.py
just publish
```

## Lesson Structure

```
lessons/<topic>/<name>/
  lesson.hs        Haskell implementation — runs via runghc
  lesson.py        Python implementation — runs via uv run python
  lesson.qmd       Quarto page (motivation + code for HTML; LaTeX imports for PDF)
  exports/
    lesson-hs.tex  generated by just export-latex — do not edit by hand
    lesson-py.tex  generated by just export-latex — do not edit by hand
```

## Authoring Guide

### `lesson.hs`

- Open with a comment block: lesson title, topic, and `Run: runghc lesson.hs`
- Define pure functions that illustrate the concept step by step
- `main` should print worked examples that confirm correctness
- **No external packages** — must run with `runghc` against the standard GHC libraries only
- Match function names and worked examples to the Python version where practical

```haskell
-- | Lesson: Modular Arithmetic
-- Topic:   foundations
-- Run:     runghc lesson.hs
module Main where

modAdd :: Int -> Int -> Int -> Int
modAdd m a b = (a + b) `mod` m

main :: IO ()
main = do
  print (modAdd 7 5 4)   -- 2
  print (modAdd 7 3 3)   -- 6
```

### `lesson.py`

- Open with a module docstring: lesson title, topic, and `Run: uv run python lesson.py`
- Functions should mirror the Haskell implementations as closely as Python allows
- `if __name__ == "__main__":` block with the same worked examples as `main` in Haskell
- Use only the standard library, or packages already declared in `pyproject.toml`
- Type-annotate all function signatures

```python
"""
Lesson: Modular Arithmetic
Topic:  foundations
Run:    uv run python lesson.py
"""


def mod_add(m: int, a: int, b: int) -> int:
    return (a + b) % m


if __name__ == "__main__":
    print(mod_add(7, 5, 4))   # 2
    print(mod_add(7, 3, 3))   # 6
```

### `lesson.qmd`

Fill in the YAML front matter and three sections:

```qmd
---
title: "Modular Arithmetic"
description: "Arithmetic on a finite cycle — the foundation of most symmetric ciphers."
categories: [foundations]
date: today
---

## Motivation

One paragraph on why this concept matters in a cybersecurity context.
Keep it concrete: name a real algorithm or attack it enables or prevents.

## Command Reference

| Recipe | What it does |
|--------|--------------|
| `just new-lesson <topic> <name>` | Scaffold `lesson.hs`, `lesson.py`, `lesson.qmd`, and `exports/` from templates |
| `just validate <path>` | Run `.hs` via `runghc` or `.py` via `uv run python` (extension auto-detected) |
| `just export-latex <lesson_dir>` | Generate both `exports/lesson-hs.tex` and `exports/lesson-py.tex` |
| `just export-latex <lesson_dir> hs` | Generate Haskell LaTeX export only |
| `just export-latex <lesson_dir> py` | Generate Python LaTeX export only |
| `just render` | Build the full site locally → `_site/` |
| `just preview` | Live-reload dev server at localhost |
| `just publish` | Deploy to GitHub Pages (manages `gh-pages` branch) |
| `just publish-gist <path>` | Create a public GitHub Gist for a single file |
